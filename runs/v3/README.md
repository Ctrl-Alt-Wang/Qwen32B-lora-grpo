## 深入分析

### 1. 两次训练不稳定 spike

**Spike 1（step 77）：轻微，自愈**
```
step 77: grad_norm=5.50, KL=0.154, reward=0.718, LR=9.82e-5
step 78: grad_norm=0.065, KL=0.046 → 完全恢复
```
单步梯度爆炸，未造成持续影响。reward 在该区间正常波动（0.70–0.75）。

**Spike 2（step 266）：较大，影响约 30 步**
```
step 266: grad_norm=4.09, KL=0.697, reward=0.775, LR=6.89e-5
step 267: KL=0.074 → 大幅回落
step 268: KL=0.189, grad=0.559 → 余震
step 269: KL=0.062 → 基本恢复
```
KL 瞬间冲至 0.697（正常值 ~0.07 的 10 倍），对应 WandB 中 step 250–350 的明显下跌段。从 step 270 起逐步恢复，step 392 后完全稳定。

**Spike 的共同特征：**
- 均为单步事件，非持续性不稳定
- 原因：v3 max_steps=672，cosine LR 在 step 77（LR≈9.8e-5）和 step 266（LR≈6.9e-5）时均较高，一旦遇到特殊 batch（高方差 reward）就容易触发大梯度

### 2. v3 中间段低于 v2 的原因

cosine LR 在相同步数处，v3（max_steps=672）的 LR 高于 v2（max_steps=448）：

| 步数 | v2 LR | v3 LR |
|------|-------|-------|
| step 56 | ~8.5e-5 | ~9.7e-5（+14%） |
| step 112 | ~6.0e-5 | ~9.1e-5（+52%） |
| step 224 | ~2.0e-5 | ~7.5e-5（+275%） |
| step 280 | ~5.5e-5 | ~6.9e-5（+25%）|

v3 中间段 LR 更高 → 策略更新幅度更大 → 震荡更剧烈。这是 cosine schedule 与更长训练周期的自然副作用，不影响最终收敛质量。

### 3. 收敛验证

后 224 步（step 448–671）四个分段均值：

```
448–503：0.765
504–559：0.759
560–615：0.761
616–671：0.761
```

均值漂移：0.765 → 0.761，差异 0.004（可忽略）。后 100 步 std=0.015 < 0.02，确认收敛。

### 4. 收敛平台 ~0.76 是该配置的性能上限

- v2 后100步：0.754
- v3 后100步：0.761
- 两者在统计误差范围内基本一致

延长训练步数（448→672）只在 step 400 后出现微弱的额外提升，增量约 +0.007。继续增加步数不会显著突破 0.76。

**突破 0.76 平台需要：**
- 增大 N_ROLLOUTS（8 或更多）→ 更精确的 advantage 估计
- 调整奖励函数权重（format vs RM score）
- 更大的 LoRA rank（r=16）→ 更强的表达能力
- 引入多样性更高的训练数据
