## 核心结论

> **v5 训练 reward 收敛平台 0.770，低于 v4 的 0.786（-0.016）。evidence_ii 评测 65.4，与 v4（65.5）基本持平（-0.1）。max_completion_length 不是当前瓶颈——v4 的生成长度增长是奖励驱动的主动选择，而非被 512 截断的结果。**

### 核心发现

1. **v4 的 414 token 均值不是被截断造成的。** v5 放开至 1024 后，模型 mean_length 反而从 v4 的 414 下降到 270-325。说明 v4 在 512 上限内自然找到了 400+ token 的最优长度，并未被截断。

2. **更长的生成空间降低了训练 reward。** v5 收敛平台（0.770）低于 v4（0.786），奖励函数对 400-500 token 长度的回答"更友好"，放开上限后模型反向收缩到更短的输出（270-325 token）。

3. **证据回答一致性因简洁化而下降。** v5 回答更短，引用数量减少，导致"证据回答一致性"从 v4 的 44.0 降至 37.3（-6.7）。

## 分析与讨论

### 为什么 max_completion_length=1024 没有带来提升？

**1. 奖励函数对长度的隐式偏好**

训练 reward 由 `0.5×格式分 + 0.5×RM分` 构成。格式分基于 PICO 关键词检测，在 400-500 token 的回答中通常已能触达满分。RM 分对过长的医学回答可能存在轻微惩罚（信息密度降低、重复内容增多）。v3/v4 通过 672 步训练建立了"400-450 token 最优"的隐式偏好，v5 从基模型重新训练，在没有这个偏好的情况下尝试更长输出，发现奖励不更高，最终收缩。

**2. 512 限制未实际生效**

v4 末期 mean_length=414，且 clipped_ratio=0 全程为零，说明 512 的上限从未真正截断任何回答。v4 的生成长度是奖励信号驱动下的主动选择，而非被迫限制的结果。这一假设被 v5 的实验直接证伪。

**3. 证据回答一致性的矛盾**

v5 回答更短→引用数量减少→证据回答一致性（44.0→37.3）下降。这与"更长回答有助于引用更多文献"的直觉一致，但讽刺的是，允许更长回答的 v5 实际上写出了更短的回答。

### 什么是真正的瓶颈？

经过五次实验（v1-v5），已系统排除以下方向：
- ✗ 训练步数（v3 已收敛）
- ✗ N_ROLLOUTS（v2 已验证 4 是合理值）
- ✗ LoRA rank（v4 r=16 边际收益有限）
- ✗ max_completion_length（v5 证明非瓶颈）

**真正未被探索且最可能有效的方向：**

**奖励函数对齐（最高优先级）**：当前 reward 对"证据引用质量"几乎没有直接信号。在 evidence_ii 评测中持续垫底的"逻辑严谨性"（33.3）和"证据回答一致性"（37.3）恰恰是奖励函数从未直接优化过的维度。加入引用一致性奖励（引用编号与声称证据类型匹配）和引用覆盖奖励（检测 [^xx-xxxx-x] 编号数量）可能是突破 65-66 分天花板的关键。
